Actividad 1

By Laura García (DevDygarcia)

¿Qué es un Dato?

Un dato es la representación simbólica de una variable, ya sea cuantitativa o cualitativa. Puede expresarse mediante números, letras o símbolos. Los datos describen hechos empíricos y, por sí solos, no tienen un significado completo.

¿Qué es información?

La información es un conjunto organizado de datos relevantes para uno o más sujetos. Representa conocimientos comunicados, compartidos o transmitidos, y constituye algún tipo de mensaje.

¿Diferencia entre dato e información?

Los datos son hechos y cifras sin contexto ni significado, mientras que la información es el resultado de analizar y dar sentido a esos datos.

¿Tipos de datos?

Los datos son hechos y cifras sin contexto ni significado, mientras que la información es el resultado de analizar y dar sentido a esos datos.

  1. Texto: Puede ser de carácter (una sola unidad o símbolo) o cadena (una sucesión de símbolos).
  2. Numérico: Incluye datos enteros (sin parte decimal) y reales (con parte decimal).
  3. Lógico: Representa valores verdaderos o falsos (booleanos)

¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear una representación visual de un sistema de información. Se comunica cómo se relacionan los datos y sus estructuras. Los modelos de datos ilustran los tipos de datos, las relaciones y la organización dentro del sistema

¿Tipos de bases de Datos según Variabilidad, Contenido y Modelo de base de datos?

Las bases de datos pueden clasificarse de diferentes maneras:

  1. Variabilidad: Pueden ser estáticas o dinámicas.
  2. Contenido: Pueden contener datos de texto, imágenes, videos, etc.
  3. Modelo de base de datos: Pueden ser relacionales, jerárquicas, de red, etc.

¿Qué es el modelo relacional?

El modelo relacional es un enfoque para organizar datos en tablas (llamadas relaciones). Cada tabla representa una entidad y sus atributos, y las relaciones se establecen mediante claves primarias y foráneas.

¿Elementos del Diagrama entidad Relacional?

En un diagrama entidad-relación, los elementos clave son:

  1. Entidades: Representan objetos del mundo real.
  2. Atributos: Características de las entidades.
  3. Relaciones: Conexiones entre entidades.

¿Qué es una entidad?

Una entidad es un objeto o concepto del mundo real que se representa en una base de datos. Por ejemplo, “Cliente” o “Producto”.

¿Qué es un atributo y sus tipos?

Un atributo es una característica de una entidad. Puede ser numérico, textual o lógico. Ejemplos: “Edad” (numérico), “Nombre” (texto), “Activo” (lógico).

¿Qué es una relación?

Una relación describe cómo se conectan dos o más entidades en una base de datos. Por ejemplo, “Cliente compra Producto”.

¿Qué son las llaves primarias, foráneas y sus características?

La llave primaria es un atributo único que identifica de manera única una entidad. La llave foránea es un atributo que se relaciona con la llave primaria de otra entidad. Ambas garantizan la integridad y coherencia de los datos.

¿Qué son las reglas del Negocio?

Las reglas del negocio son restricciones o directrices que definen cómo deben operar los datos en una organización. Por ejemplo, políticas de acceso o validaciones.

¿Qué es la cardinalidad o mapeo?

La cardinalidad describe la cantidad de instancias de una entidad que pueden estar relacionadas con instancias de otra entidad en una relación. Hay varios tipos de cardinalidad:

  1. Uno a uno (1:1): Cada instancia de una entidad se relaciona con exactamente una instancia de otra entidad, y viceversa.
  2. Uno a muchos (1:N): Cada instancia de una entidad se relaciona con una o más instancias de otra entidad, pero cada instancia de la segunda entidad se relaciona con solo una instancia de la primera entidad.
  3. Muchos a muchos (N:M): Varias instancias de una entidad se relacionan con varias instancias de otra entidad. Esto se logra mediante una tabla intermedia o de unión.
